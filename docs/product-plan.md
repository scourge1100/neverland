# Neverland 제품 방향

## 목표

Neverland를 글과 프로젝트가 서로 연결되는 포트폴리오형 개인 블로그로 만든다.

- 글은 단순한 일지가 아니라 문제 해결 과정과 배운 점을 보여준다.
- 프로젝트는 결과물뿐 아니라 배경, 역할, 판단과 성과를 보여준다.
- Markdown은 작성 원본과 변경 이력을 함께 관리한다.
- 공개 화면은 독자가 탐색하기 좋은 블로그 경험을 제공한다.
- 공개 운영에는 상시 서버나 데이터베이스를 사용하지 않는다.

## 정보 구조

1. Home: 소개, 최근 글, 대표 프로젝트, About 요약
2. Writing: 전체 글, 카테고리, 태그, 연도별 아카이브
3. Projects: 대표 프로젝트 목록과 case study
4. About: 소개, 경력, 기술, 연락처
5. Search: 글과 프로젝트 통합 검색
6. RSS: 공개 글 구독

## 콘텐츠 원칙

### Writing

- 제목과 한 줄 요약
- 카테고리와 태그
- 작성일과 수정일
- 연관 프로젝트
- 문제, 판단, 해결, 배운 점

### Project

- 프로젝트 배경과 해결하려던 문제
- 기간과 담당 역할
- 제약과 주요 의사결정
- 구현 내용과 사용 기술
- 결과와 배운 점
- 관련 글과 외부 링크

## 구현 단계

### Phase 1 — Identity and home

- 포트폴리오형 블로그 홈
- 최근 글과 대표 프로젝트 영역
- About 요약과 모바일 대응

### Phase 2 — Content pipeline

- Markdown frontmatter 기반 콘텐츠 수집
- 글과 프로젝트용 메타데이터 규칙
- GitHub push 기반 자동 빌드와 배포

### Phase 3 — Discovery

- 글 목록과 상세 화면
- 카테고리, 태그, 연도별 아카이브
- 통합 검색, RSS, sitemap, SEO와 공유 카드

### Phase 4 — Portfolio depth

- 프로젝트 case study
- 경력과 기술 영역
- GitHub 저장소와 릴리스 활동 연동

### Phase 5 — Operations

- Azure Static Web Apps Free 플랜 배포
- Git 저장소 기반 콘텐츠 백업과 변경 이력
- 빌드 검사, 링크 점검과 배포 상태 확인

## 현재 결정이 필요한 콘텐츠

- 공개할 이름과 한 줄 소개
- 대표 프로젝트 3~5개
- 공개할 GitHub 계정과 저장소 범위
- 연락 수단
- 선호 도메인 또는 무료 Azure 기본 주소 사용 여부
